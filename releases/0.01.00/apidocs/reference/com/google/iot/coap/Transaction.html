public interface

Transaction

com.google.iot.coap.Transaction

Class Overview

Interface representing an outbound transaction and keeping track of its pending response.

Transaction objects keep track of an outbound request and the associated inbound response. Usually this is a one-to-one relationship, but there are a few cases where transactions can have more than one inbound response, such as multicast requests or observing requests.

Transactions could be thought of as similar to a special com.google.common.util.concurrent.ListenableFuture returning a response Message, with a fair amount of additional behaviors for good measure.

public abstract void cancel ()

Cancels and invalidates this transaction. Any pending or later calls to getResponse() will throw a CancellationException. Cancelling a transaction will permanently invalidate it: it cannot be resumed by calling restart().

If the transaction is not active, calling this method does nothing.

public abstract Message getResponse (long timeout)

Accessor for the response, with timeout. This method will block until a response has been received or until some other condition has occurred, indicated by the exception thrown.

If a response has already been received, the most recently received response is immediately returned without blocking. If the underlying transaction has been cancelled or stopped for a reason other than receiving a response message, an exception is immediately thrown.

Parameters
timeout the maximum amount of time to block execution, in milliseconds.
Returns
  • the most recently received response message. If this is a multicast transaction that has timed out, the returned value will be null.
Throws
InterruptedException if this thread has been interrupted.
HostLookupException if the host name lookup failed.
IOException if there was an underlying IOException
TimeoutException if there was no response within the time frame described by timeout. This will not be thrown if this transaction is a multicast transaction.
CancellationException if this transaction has been cancelled

public abstract Message getResponse ()

Accessor for the response. This method will block until a response has been received or until some other condition has occurred, indicated by the exception thrown.

Although this method has no timeout parameter, it may still throw a TimeoutException if this is not a multicast transaction and the method blocks for a time period larger than given by getCoapExchangeLifetimeMs().

If a response has already been received, the most recently received response is immediately returned without blocking. If the underlying transaction has been cancelled or stopped for a reason other than receiving a response message, an exception is immediately thrown.

  • the most recently received response message. If this is a multicast transaction that has timed out, the returned value will be null.

public abstract boolean isActive ()

Indicates if the transaction is active or not. An active transaction is one that is either in the process of transmitting the request or is still actively waiting for a response. Most transactions become inactive after receiving a a single non-empty response, but multicast and observing transactions will generally remain active until cancelled.

Returns
  • true if the transaction is active, false otherwise.

public abstract boolean isFinishedAfterFirstResponse ()

Indicates if the transaction is considered finished after receiving the first response. This will be the case for observing transactions and multicast transactions. Note that an acknowledgement packet is not considered a response.

public abstract void registerCallback (Executor executor, Transaction.Callback cb)

Registers a callback class with the transaction, to be executed with the given executor.

If this transaction is no longer active then the appropriate methods on the callback class will be immediately scheduled for execution with the given executor.

Parameters
executor the executor object to use to call the methods on the given callback class
cb the callback class instance to register

public abstract void restart ()

Sends the request, making the transaction active. This method will restart an inactive transaction, unless it was cancelled by calling cancel().

This method isn't usually called because the method send() will call it automatically. However, if you finish building the request using prepare() instead of send(), then you will need to call this method in order to actually send the request.

Throws
CancellationException if this transaction has been cancelled.

public abstract void unregisterCallback (Callback cb)

Unregisters a previously registered callback class from this transaction. If the given callback instance was not previously registered or is null, then this method does nothing.

Parameters
cb the callback class instance to unregister
